# Jumping Slime

## Group Members
Aisling Li, Zora Shi

## Brief Description
A two-player platform jumping game played on the same device, where players compete using the keyboard to reach the goal platform first.

## Game Overview / UI Concept
The game features a colorful 2D platform environment inspired by platform jumping games. Each player controls a slime character that can move and jump across floating platforms. A goal platform with a flag is placed somewhere in the level. Players start on opposite sides and must navigate obstacles to reach the goal.

The interface includes:
- Two slime characters (1P and 2P)
- Multiple platforms at different heights
- A goal platform marked with a flag
- A score system showing progress across rounds
- Simple UI screens (menu, gameplay, win screen)

## Why This Project Is Interesting
This project is interesting because it combines simple mechanics with competitive gameplay. While the controls are easy to learn, the physics (jumping, gravity, and movement) create a skill-based experience that makes each round dynamic and unpredictable.

It is also a fun way to explore concepts in object-oriented programming, such as game state management, collision detection, and event handling. Additionally, designing levels and balancing difficulty adds a creative aspect beyond just coding.

Most importantly, it is a local two-player game, which makes it more engaging and interactive compared to single-player projects.

## Minimum Viable Product (MVP)
The initial version of the game will include:
- Two controllable players (keyboard input)
- Basic movement (left/right) and jumping with gravity
- A single level with a few platforms
- Collision detection between players and platforms
- A goal platform that determines the winner

This version focuses on making the core gameplay functional before adding additional features.
